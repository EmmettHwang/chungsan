### 3. `bump-version.sh` - 버전 업데이트 자동화 스크립트

**"버전 올려" 명령어 실행 시 자동 프로세스:**

#### 1️⃣ 버전 번호 증가
- README.md에서 현재 버전 추출
- 버전 업데이트 유형 선택:
  - **Patch**: v5.6.9 → v5.6.10
  - **Minor**: v5.6.9 → v5.7.0
  - **Major**: v5.6.9 → v6.0.0
- 타임스탬프 자동 추가 (YYYYMMDDHHMM 형식)

#### 2️⃣ README.md 정리
- 변경사항 입력 프롬프트
- 자동으로 README.md 상단에 변경 이력 추가
- 날짜 및 버전 정보 업데이트

#### 3️⃣ 캐시 버스팅 정리
- `frontend/app.js`의 `APP_VERSION` 업데이트
- `frontend/service-worker.js`의 `CACHE_VERSION` 업데이트
- 클라이언트 캐시 자동 무효화

#### 4️⃣ Git 커밋
- 모든 변경사항 스테이징
- 커밋 메시지 자동 생성:
  ```
  chore: bump version to v버전번호
  
  변경사항 요약
  
  - 버전 업데이트: v이전버전 → v새버전
  - README.md 업데이트
  - 캐시 버스팅 적용
  ```

#### 5️⃣ Git Push
- 현재 브랜치에 자동 푸시

#### 6️⃣ Main 브랜치에 병합
- main 브랜치로 체크아웃
- 최신 상태로 pull
- 현재 브랜치 병합 (--no-ff 옵션)
- main 브랜치 푸시

#### 7️⃣ 작업 브랜치로 복귀
- 원래 브랜치로 자동 복귀
- 완료 메시지 출력

**실행 방법:**
```bash
cd /home/user/webapp
./bump-version.sh
```

**대화형 프롬프트:**
1. 버전 업데이트 유형 선택 (1: Patch / 2: Minor / 3: Major)
2. 주요 변경사항 입력
